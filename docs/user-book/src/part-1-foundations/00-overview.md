# Part I — Foundations

This part is intentionally code-light. Its goal is to give you enough vocabulary
and intuition to read the guided tour in Part II without getting lost, and enough
references to dig deeper when you want to.

Each section is a conceptual summary — a page or two that captures the core idea
and names the key results. Where an algorithm or derivation deserves more than a
summary, a callout box points to the relevant appendix.

**You do not have to read this part before Part II.** If you learn better by
doing, start with the tour and come back here when a term needs grounding. The
cross-references work in both directions.
