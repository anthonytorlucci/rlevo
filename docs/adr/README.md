# Architectural Decision Records

Immutable records of the architectural decisions behind `rlevo`. Once accepted,
an ADR is not edited — a later decision supersedes it, and the superseded record
is annotated rather than deleted. Read these for the *why* behind the crate
boundaries and trait design before proposing structural changes.

When you make an architectural decision, add a new numbered file here following
the existing format (`Status` / `Context` / `Decision` / `Consequences` /
`Alternatives considered` / `References`).

> Drafts and in-flight proposals live in the maintainer's working notes; an ADR
> lands here only once accepted. The repo copy is canonical.

| [0026](0026-shared-config-validation-convention.md) | Introduce a standalone `Validate` trait (`fn validate(&self) -> Result<(), ConfigError>`) + structured allocation-free `ConfigError`/`ConstraintKind` in a new `rlevo_core::config` module — one shared config-validation convention for the workspace's 87 `*Config` types. Resolves the standing `rules.md §4` panic-vs-`Result` contradiction: `Deserialize`-able configs are user data, so an **assembled** config validates to `Result` while a single documented **builder setter** (`with_capacity`/`with_alpha`) may still panic. `Default`s must pass their own `validate()`. Purely additive to `rlevo-core`; adoption of the ~85 remaining configs is deferred to the per-crate issues (#102/#103, #137–139, #164–166) this unblocks. | [0027](0027-bounds-newtype-for-closed-ranges.md) | Add a `Bounds` inclusive-range newtype (`rlevo_core::bounds`) that is **valid by construction** — the whole invariant is `lo <= hi`, rejecting `lo > hi` and `NaN` while allowing a single-point `lo == hi` and a one-sided infinite `[0.7, ∞)` healthy interval — making the `f32::clamp` panic and the `x.max(lo).min(hi)` silent-collapse hazard unrepresentable wherever a `Bounds` is held, not just at the config boundary. Complements ADR 0026: a `Bounds` field is self-validating, so it **removes** the paired `config::ordered("bounds", …)` line (~22 sites) rather than replacing `Validate`. Validated `Deserialize` via `#[serde(try_from)]`; dedicated `Copy` `BoundsError` (constructor has no field name). Migrates every `(f32,f32)` range field in `rlevo-evolution` (all `bounds`) and `rlevo-environments` (`action_clip`, `HealthyCheck` ranges, `clip_contact_cost`) and folds mountain-car's `min/max` scalar pairs into `Bounds` (breaking serde shape, alpha). Closes #99/#140/#196 and fixes three previously-unguarded sites (four local-search params, `HealthyCheck::state_range`, `clip_contact_cost`). |
| [0028](0028-tensor-batch-conversion-seam.md) | Make the host-only flat-`f32` **row-writer the required primitive** of `rlevo_core::TensorConvertible` (`row_shape()` + `write_host_row()`); `to_tensor` becomes a **derived** default (do not override) so the single-item and batch layouts share one definition; new free fn `stack_to_tensor<R, BR>` stages a whole batch as one host buffer and issues a **single** `Tensor::from_data` upload (the sole library-side `BR = R + 1` assert — stable Rust cannot express `R + 1` in types). Burn 0.21 audit: no built-in single-upload batcher exists (Burn's own `Batcher`/MNIST idiom is N transfers + `cat`), so the helper is additive, not redundant. First consumer: `sample_batch` in RL `memory.rs` — obs/actions/next-obs via `stack_to_tensor` (one upload each, was N); rewards stay rank-1 `[batch]` via the row-writer + one `from_floats` (never the helper, which would yield `[batch, 1]`). **Breaking** trait change migrating all ~27 impls in one PR (alpha); `from_tensor` untouched; inverse batch decode and `Kind` (`Int`/`Bool`) generalisation explicitly deferred. Implements #195 P2; P1 (the `unwrap_or_default` footgun) lands as a `docs/rules.md` convention + CI grep guard, not a sweep. |
| [0029](0029-host-rng-seeding-convention.md) | Codify the **host-RNG seeding convention** (already followed in `rlevo-evolution`) as a `docs/rules.md` §8 rule and bring the two lagging crates into line. Environments **own a persistent RNG seeded once at construction**; `reset()` advances the stream (successive episodes differ) instead of re-seeding from `config.seed` each call — fixing #104 across all genuinely-stochastic envs (locomotion, classic-control, `pixel_grid`, `dynamic_obstacles`). Deterministic replay moves to an inherent `reset_with_seed(seed)`; run-level reproducibility is still guaranteed by the construction seed. The **bandit family** is brought into compliance too: the problem instance (arm-means / phases / context) is sampled **once at construction** and preserved across resets (`NonStationaryBandit` restores its drift baseline), so reward noise varies per episode; `AdversarialBandit` is genuinely deterministic (pure-function reward) and drops its now-unused RNG field. The deterministic `_rng` grids are no-ops and untouched. Host-sampling stays `seed_stream`/`SeedStream`-only (`B::seed` + `Tensor::random` banned — process-wide RNG mutex races parallel tests). Unifies PPO/PPG RNG trait bounds onto `R: Rng + ?Sized` (the `&mut dyn Rng` was never object-safety-load-bearing). **Behavioural / semver-relevant** env `reset()` change (alpha); otherwise additive. Extends ADR 0016/0017. Implements #197; fixes #104. |
| [0030](0030-permutation-tensorgenome-and-population-nonempty-invariant.md) | Close the three #158 core type-safety gaps in `rlevo-evolution`. **`Permutation` implements `TensorGenome`** (`Tensor<B, 2, Int>`) now — a permutation is genuinely rectangular, unlike `Tree` — so `Population<B, Permutation>` type-checks and gains a `new_permutation` constructor (its per-row bijection invariant stays **unchecked**, mirroring `Binary`/`Integer` unchecked values; the ACO operators remain a `todo!()` stub). The **`TensorGenome::Tensor` GAT gains `+ Send + Sync`** (was `Clone + Debug` only) so `Population<B, K>: Send + Sync` is provable under the `Strategy: Send + Sync` contract — a straight bug fix under ADR 0022. Extends ADR 0022; reuses ADR 0026. Fixes #158. |
